# Project Scope

> Generated by setup wizard on 2026-03-29. Updated as priorities shift.

## In Scope — v1 (Hackathon Demo)

### P0 — Must Work for Demo
- `pacifica init --testnet` — interactive onboarding wizard (60s to first trade)
- Pacifica SDK wrapper — REST + WebSocket client for all exchange operations
- `pacifica scan` — live market overview with real-time WebSocket data
- `pacifica trade buy/sell` — order placement with TP/SL
- `pacifica orders` — list and cancel open orders
- `pacifica positions` — open positions with live PnL
- MCP server with 6-10 core tools (markets, orders, positions, account)
- Agent guardrails — config, kill switch, spending limits, audit trail
- `.pacifica.yaml` config file format

### P1 — High Value, Build If Time
- Position heatmap visualization (`pacifica heatmap`)
- Funding rate scanner (`pacifica funding`)
- PnL journal — auto-logging trades to local JSON
- Smart orders — trailing stop + partial take-profit
- Claude Code Skills (5 slash commands)
- Agent-readable market intelligence (M10 — 5 MCP tools, alerts CLI)
- Funding rate arbitrage bot (M11 — `pacifica arb` with 7 subcommands, 6 MCP tools)
- Pacifica Intelligence Layer (M11 — capture, pattern engine, web UI, Elfa social signals)
- Web intelligence dashboard (M12 — read-only overlay, no trading execution)

### P2 — Only If Day 2 Goes Well
- Event hooks system (7 event types, YAML config)
- TradingView webhook bridge
- Smart orders — DCA entry, time stop, breakeven auto
- Journal export (CSV/JSON)

## Out of Scope — v1 (Possible Later)

- Mainnet mode with real money (testnet only for hackathon)
- Portfolio analytics / advanced reporting
- Multi-exchange unified order routing
- Backtesting engine
- Strategy marketplace
- Social features / copy trading
- Mobile companion app
- Historical chart rendering in terminal
- Order book depth visualization

## Never In Scope

- Hosted server / cloud deployment (runs locally only)
- User accounts or authentication system (API key only)
- Traditional database (file-based storage only)
- Telemetry or analytics collection
- Custodial key management
- Light mode
- Order execution from the web dashboard (deep links to app.pacifica.fi only)
