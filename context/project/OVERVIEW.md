# Project Overview

> Generated by setup wizard on 2026-03-29.

## What It Is

Pacifica CLI is an agent-native trading terminal for Pacifica DEX. Three deliverables from one codebase:

1. **CLI (TUI)** — Rich terminal UI with live market data, order management, position tracking, ASCII-art heatmaps. Installed via `npm install -g pacifica-cli`, runs as `pacifica <command>`.
2. **MCP Server** — Model Context Protocol server exposing 20+ tools so AI agents (Claude, etc.) can read markets, place orders, manage positions on Pacifica programmatically.
3. **Claude Code Skills** — 5 slash commands (`/pacifica scan`, `/pacifica trade`, `/pacifica positions`, `/pacifica heatmap`, `/pacifica agent`) for agent-assisted trading workflows.

## The Problem

Full-time perps traders who live in the terminal (like "Minh" — 27, HCMC, trades from a Linux laptop, 3-5 positions across multiple exchanges) are stuck in a painful loop:

1. Copy data from Pacifica's web UI
2. Paste into Claude for analysis
3. Get the recommendation
4. Go BACK to the browser
5. Manually enter the order, set TP/SL
6. Hope nothing was fat-fingered

By the time the loop completes, the opportunity has moved. The web UI is designed for casual users — power traders need speed, information density, and agent integration.

## The Solution

A terminal-first trading interface with three layers:

- **Layer 1 — CLI (TUI):** Minh uses it directly. Rich terminal UI with live prices, positions, order entry.
- **Layer 2 — MCP Server:** Claude uses it. Standard MCP tools so any AI agent can read markets and execute trades.
- **Layer 3 — Skills:** Slash commands that combine both — Minh talks to Claude, Claude uses MCP tools, results render in terminal.

Everything runs locally. No hosted frontend to geo-block. No middleware logging trading patterns. No third-party holding API keys.
