# CI/CD Pipeline

> Generated by setup wizard on 2026-03-29.

## Pipeline Overview

For a 3-day hackathon, CI/CD is minimal:

- **On push to main:** TypeScript type-check (`tsc --noEmit`) + build (`tsup`)
- **Pre-commit (optional):** Lint + type-check
- **No automated deployment** — manual `npm publish` when ready

## Deploy Process

### npm Publish (Post-Hackathon)
1. Bump version in `package.json`
2. `pnpm build`
3. `npm publish`

### GitHub Release (Post-Hackathon)
1. Tag the release: `git tag v1.0.0`
2. Build binaries: `pkg dist/cli.js -t node18-linux,node18-macos,node18-win`
3. Upload to GitHub Release

## Secrets in CI

No CI secrets needed during hackathon. Post-hackathon:
- `NPM_TOKEN` — for automated npm publishing
