# Infrastructure

> Generated by setup wizard on 2026-03-29.

## Environments

| Environment | Purpose | How to Use |
|-------------|---------|-----------|
| **Local (testnet)** | Development and demo | `pacifica init --testnet` or `pacifica --testnet <cmd>` |
| **Local (mainnet)** | Production trading | `pacifica init` → select Mainnet (post-hackathon) |

No hosted environments. Everything runs on the trader's machine.

## Distribution

| Channel | Method |
|---------|--------|
| **npm** | `npm install -g pacifica-cli` — primary distribution |
| **GitHub Releases** | Prebuilt binaries via `pkg` or `bun build --compile` (post-hackathon) |
| **Source** | `git clone` + `pnpm install` + `pnpm build` |

## Storage

All data is local filesystem:
- Config: `.pacifica.yaml` in user's home or cwd
- Runtime data: `.pacifica/` directory
- No cloud storage, no S3, no CDN

## External Dependencies

| Service | SLA Needed | Fallback |
|---------|-----------|----------|
| Pacifica REST API | Required for all operations | Retry with backoff |
| Pacifica WebSocket | Required for live data | Fall back to REST polling every 2s |
