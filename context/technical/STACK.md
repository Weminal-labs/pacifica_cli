# Tech Stack

> Generated by setup wizard on 2026-03-29.

## Language & Runtime

| Choice | Why |
|--------|-----|
| **TypeScript** | Type safety for trading logic (numbers matter when money is involved). Best MCP SDK support. npm ecosystem for CLI tooling. |
| **Node.js** | Runtime for TypeScript. Async I/O for WebSocket + REST. Required by Ink. |

## CLI Layer

| Choice | Why |
|--------|-----|
| **Ink (React for CLI)** | Component model for TUI. Live updates via React state. Rich rendering — colors, tables, progress bars. Handles terminal resizing. |
| **commander** | CLI argument parsing. Lightweight, good TypeScript support. Subcommand model fits `pacifica trade buy/sell` pattern. |
| **@inquirer/prompts** | Interactive prompts for `pacifica init` wizard. Modern, promise-based, composable. |
| **chalk** | Terminal color output for non-Ink commands (pipe-friendly output). |

## MCP Layer

| Choice | Why |
|--------|-----|
| **@modelcontextprotocol/sdk** | Official MCP SDK. Standard protocol for AI agent tools. Works with Claude Code natively. |

## Network

| Choice | Why |
|--------|-----|
| **ws** | WebSocket client for real-time market data from Pacifica. Lightweight, battle-tested. |
| **node built-in fetch** | REST API calls to Pacifica, Binance, Bybit. No extra HTTP client dependency needed. |
| **fastify** | Local HTTP server for TradingView webhook bridge (P2). Lightweight, typed, fast. |

## Config & Storage

| Choice | Why |
|--------|-----|
| **yaml** (npm package) | Parse `.pacifica.yaml` config. Human-readable, trader-friendly. |
| **Local JSON files** | Append-only logs for journal, agent trail, smart orders, hooks. Simple, inspectable. |

## Build & Dev

| Choice | Why |
|--------|-----|
| **tsup** | Fast TypeScript bundling. Single entry point → `dist/cli.js`. |
| **pnpm** | Fast, disk-efficient package manager. |
| **vitest** | Testing framework if time allows. Fast, TypeScript-native. |

## Infrastructure

| Choice | Why |
|--------|-----|
| **None** | Runs locally on trader's machine. No hosting. Published to npm for distribution. |

## External APIs

| API | Auth | Purpose |
|-----|------|---------|
| **Pacifica REST API** | API key + secret | Markets, orders, positions, account |
| **Pacifica WebSocket** | API key + secret | Real-time ticker, fills, position updates |
| **Binance Public API** | None (public) | Funding rate comparison for arb scanner |
| **Bybit Public API** | None (public) | Funding rate comparison for arb scanner |

## Rationale

This stack optimizes for:
1. **Speed of development** — 3-day hackathon, solo developer
2. **Type safety** — Trading logic cannot have type errors
3. **MCP ecosystem** — TypeScript has the best MCP SDK support
4. **Terminal experience** — Ink gives us React's component model in the terminal
5. **Zero hosting cost** — Everything local, free tiers only
