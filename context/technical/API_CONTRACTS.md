## Pacifica REST API

### Authentication

All authenticated requests use **Ed25519 wallet signatures**:
1. Recursively sort request payload keys
2. Compact JSON serialize (no whitespace)
3. Sign with Ed25519 using the private key
4. Base58-encode the signature
5. Include public key and signature in request headers/body

Libraries: `tweetnacl` for signing, `bs58` for Base58 encoding/decoding.

## Pacifica DEX Proxy Routes (M12)

All routes proxy to https://test-api.pacifica.fi/api/v1. GET only (no auth required).

### GET /api/pacifica/account/:address
Returns master account equity and balance details.
```json
{
  "address": "string",
  "balance": "string",
  "account_equity": "string",
  "available_to_spend": "string",
  "available_to_withdraw": "string",
  "positions_count": "number",
  "orders_count": "number",
  "fee_level": "number",
  "maker_fee": "string",
  "taker_fee": "string",
  "cross_mmr": "string"

```
