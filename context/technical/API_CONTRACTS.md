## Pacifica REST API

### Authentication

All authenticated requests use **Ed25519 wallet signatures**:
1. Recursively sort request payload keys
2. Compact JSON serialize (no whitespace)
3. Sign with Ed25519 using the private key
4. Base58-encode the signature
5. Include public key and signature in request headers/body

Libraries: `tweetnacl` for signing, `bs58` for Base58 encoding/decoding.

### Market Data (Public)

| Method | Endpoint | Purpose |
|--------|----------|---------|
| GET | `/api/v1/info` | List all markets and their metadata |
| GET | `/api/v1/info/prices` | Current prices for all markets |
| GET | `/api/v1/book` | Order book depth for a market |
| GET | `/api/v1/trades` | Recent trades for a market |
| GET | `/api/v1/funding_rate/history` | Historical funding rates |

### Account (Authenticated)

| Method | Endpoint | Purpose |
|--------|----------|---------|
| GET | `/api/v1/account` | Account balance and margin |
| GET | `/api/v1/positions` | List open positions |
| GET | `/api/v1/orders` | List open orders |
| GET | `/api/v1/trades/history` | Trade fill history |

### Order Management (Authenticated)

| Method | Endpoint | Purpose |
|--------|----------|---------|
| POST | `/api/v1/orders/create_market` | Place market order |
| POST | `/api/v1/orders/create` | Place limit order |

| POST | `/api/v1/positions/tpsl` | Set take-profit / stop-loss on a position |
