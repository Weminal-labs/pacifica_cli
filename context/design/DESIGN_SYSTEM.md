# Design System

> Generated by setup wizard on 2026-03-29. Terminal-only — no web/mobile design.

## Direction & Tone

**Fast, dense, professional.** Like Bloomberg Terminal meets `htop`.

| Quality | In Practice |
|---------|-------------|
| **Fast** | Data updates real-time. Commands respond instantly. No loading spinners > 500ms. |
| **Dense** | Maximum information per screen. No wasted whitespace. Multiple data points visible without scrolling. |
| **Professional** | Muted color palette. No emojis in data. No "cute" error messages. |
| **Trustworthy** | Every action confirms what happened. Every trade shows fill price, fees, margin impact. |

## Terminal Color Palette

All colors use ANSI 256 or true-color where supported, with fallback to basic 16 colors.

| Role | Color | ANSI | Usage |
|------|-------|------|-------|
| **Profit / Positive** | Green | `\x1b[32m` | PnL positive, price up, success messages |
| **Loss / Negative** | Red | `\x1b[31m` | PnL negative, price down, errors, DANGER alerts |
| **Warning** | Yellow | `\x1b[33m` | Warnings, WATCH alerts, approaching limits |
| **Label / Header** | Cyan | `\x1b[36m` | Column headers, section titles, metadata labels |
| **Muted / Secondary** | Dim/Gray | `\x1b[2m` | Timestamps, IDs, less important values |
| **Emphasis** | Bold White | `\x1b[1m` | Key values, market names, current price |
| **Background** | Terminal default | — | Dark background assumed (no light mode) |

## Typography

Terminal monospace font — no choice here. Design for:
- **80-column minimum** width (standard terminal)
- **120-column recommended** for full tables
- Detect terminal width and adapt table column widths

## Table Formatting

- Right-align numbers
- Left-align text (market names, statuses)
- Use `═` for major section dividers
- Use `─` for subsection dividers
- Use `│` for column separators only when needed for density
- Pad columns with 2 spaces minimum
- Truncate long values with `...` rather than wrapping

## Number Formatting

| Type | Format | Example |
|------|--------|---------|
| Price (< $1) | 4-6 decimals | `$0.0045` |
| Price ($1-$999) | 2 decimals | `$178.50` |
| Price (> $1000) | Comma-separated, 0-2 decimals | `$97,200` |
| Percent | 1 decimal with sign | `+3.2%`, `-0.8%` |
| Volume/OI | K/M abbreviation | `$890K`, `$2.1M` |
| PnL | Sign prefix, 2 decimals | `+$21.00`, `-$85.00` |
| Funding rate | 3 decimals percent | `0.012%` |

## Heatmap Bars

```
░ — empty / outside range
█ — risk zone (between entry and SL, or entry and liq)
▓ — profit zone (between entry and TP)
▼ — current price marker
```

## Status Indicators

| Symbol | Meaning |
|--------|---------|
| `+` | Positive/profit |
| `-` | Negative/loss |
| `*` | Active/running |
| `!` | Warning |
| `x` | Error/failed |
| `>` | Actionable signal |

## Hard Nos

- No light mode
- No ASCII art logos or banners
- No animated transitions
- No emojis in data tables (exception: whale marker in alerts)
- No "cute" error messages
- No interactive TUI for core commands — keep pipe-friendly
- No web dashboard companion
