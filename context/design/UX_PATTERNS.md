# UX Patterns

> Generated by setup wizard on 2026-03-29. Terminal-only patterns.

## Navigation

Commands follow a `pacifica <noun> [verb] [args] [--flags]` pattern:

```
pacifica scan                           # noun only = default view
pacifica trade buy ETH-PERP 0.5         # noun verb args
pacifica positions close ETH-PERP       # noun verb args
pacifica orders cancel abc123           # noun verb args
pacifica agent status                   # noun verb
pacifica smart trailing ETH-PERP --distance 2%  # noun verb args --flags
```

Users navigate by typing commands, not by cursor/arrow keys in menus. Core commands are pipe-friendly — output can be piped to `grep`, `jq`, `awk`.

Exception: `pacifica init` uses interactive prompts (arrow keys, text input) for one-time onboarding.

## Loading States

- **WebSocket connecting:** Show `Connecting...` in status line. Don't block the command.
- **REST API call:** Show inline spinner (`*`) next to the action being performed. Max 500ms before showing.
- **Reconnecting:** Show `Reconnecting...` in status bar. Auto-retry with backoff.
- **Long operation:** Show progress text. Never show a raw loading spinner with no context.

```
* Placing order...
* Connecting to Pacifica WebSocket...
! Reconnecting... (attempt 3/5)
```

## Empty States

Every list command handles zero items gracefully:

```
# No positions
No open positions.

# No orders
No open orders.

# No smart orders
No smart orders active. Use `pacifica smart --help` to get started.

# No journal entries
No trades recorded today. Place your first trade with `pacifica trade --help`.

# No hooks configured
No hooks configured. Add hooks in .pacifica.yaml under the `hooks:` section.
```

## Error Handling

### User-Facing Errors

Errors are one clear sentence. No stack traces. No raw API responses.

```
# Bad
Error: {"code": 400, "message": "insufficient_margin"}

# Good
Not enough margin for this order. You need $500 but only have $320 available.

# Bad
TypeError: Cannot read properties of undefined (reading 'price')

# Good
Could not fetch price for XYZ-PERP. This market may not exist on Pacifica.
```

### Error Categories

| Category | Format | Action |
|----------|--------|--------|
| Input validation | `Invalid: [what's wrong]. [how to fix]` | Don't send to API |
| API error | `[Plain English explanation]. [Suggestion]` | Translate API error |
| Connection error | `Cannot reach Pacifica. Check your internet connection.` | Retry with backoff |
| Config error | `Config issue: [what]. Run pacifica init to fix.` | Point to fix |
| Guardrail rejection | `Agent blocked: [reason]. Remaining daily budget: $X.` | Show remaining budget |

### Race Conditions

- "Order already filled" → `This order was already filled. Run pacifica positions to see your position.`
- "Position already closed" → `This position is already closed. Run pacifica journal to see the result.`

## Confirmations

Actions that spend money or close positions show a confirmation:

```
Placing order: BUY 0.5 ETH-PERP at market, 5x leverage
  TP: $4,000  SL: $3,700
  Estimated cost: $384.20

Confirm? [Y/n]
```

Auto-confirm with `--yes` flag for scripting.

Positions > $1,000 always require confirmation. Agent orders above `require_confirmation_above` threshold require human confirmation.

## Success Messages

After every action, confirm what happened:

```
Order filled: BUY 0.5 ETH-PERP @ $3,842.50
  Leverage: 5x  |  Fees: $1.15  |  Margin used: $384.25
  TP: $4,000  |  SL: $3,700
```

```
Position closed: ETH-PERP LONG
  Exit price: $3,900  |  PnL: +$28.75 (+0.75%)
  Duration: 2h 15m  |  Fees: $1.17
```

## Real-Time Updates

Live commands (`pacifica scan`, `pacifica heatmap`) use Ink's React state model:
- WebSocket ticks update component state
- Ink re-renders only changed cells
- Terminal flicker avoided by batch updates
- Status bar shows connection state and last update time

## Keyboard Shortcuts (Live Views Only)

| Key | Action |
|-----|--------|
| `q` | Quit live view |
| `r` | Force refresh |
| `?` | Show help overlay |
